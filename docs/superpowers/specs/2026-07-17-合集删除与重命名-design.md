# 合集删除与重命名命令设计

> 日期：2026-07-17
> 状态：已获批，待用户审阅
> 范围：在现有 `/collection create` 基础上新增 `/collection delete` 与 `/collection rename` 两个子命令

---

## 1. 背景与目标

`/collection create <名称>` 已在上个提交（`fc1c2f7`）实现，走 IndexManager Write Worker 在写锁内创建或登记 `memes/` 一级普通目录并写入 `meme_collection`。本次补齐合集生命周期的删除与重命名两个子命令，使命令组完整。

底层 `MetadataStore` 已有 `create_collection` 与 `delete_collection_and_reset_scopes`（只允许删空合集，事务内回退所有 ChatScope 到 0），但没有 `rename_collection`；且 `delete_collection_and_reset_scopes` 不碰文件系统目录。

## 2. 已确认的决策

| 决策点 | 结论 |
|---|---|
| 范围 | 只补 `delete` + `rename`，`create` 不动 |
| 删除语义 | 只允许删**空合集**，非空拒绝 |
| 空目录处理 | 写锁内 `rmdir` 删除空目录 |
| 重命名语义 | 改 `name` + 重命名目录 + 更新该合集所有 `image_path` 首段；`collection_id` 不变，chroma 不动，ChatScope 不受影响 |
| 命令格式 | `/collection delete <编号|名称>`、`/collection rename <旧编号|名称> <新名称>` |
| 确认流程 | **不需要确认**，校验通过后直接执行（与 `create` 一致） |
| 目标解析 | 旧合集接受编号或名称（同 `/switch`，编号优先、名称兜底）；新名称必须是全新名称 |
| 删除 rmdir 失败 | 先 rmdir 再删 DB，rmdir 失败时 DB 未动，无需回滚 |
| 写编排 | 方案 A：复用现有 Write Worker + 字典派发，新增两个 WriteOp |

## 3. 命令接口与插件层

### 3.1 命令格式

```
/collection create <名称>                    # 已有，不动
/collection delete <编号|名称>               # 新增，直接执行
/collection rename <旧编号|名称> <新名称>     # 新增，直接执行
```

三个子命令收在同一个 `bot/plugins/collection.py` 的 `collection_cmd` 里按子命令派发，不拆成独立插件文件。

### 3.2 权限与会话

- 权限：授权用户 + 仅私聊（组 A）；群聊 @bot 回复"此命令仅限私聊使用"；非授权用户静默忽略并记日志。
- 会话互斥：`session_manager.activate_chat(scope, "collection", matcher)`，失败回复"已有命令在处理中，请先 /cancel"；`finally` 里 `deactivate_chat`。
- **无确认等待态**：不创建 selection、不创建 timeout task、不走 `got("confirm")`；不涉及 `/cancel`/`/help` 旁路分支。插件层是"校验 → 调 IndexManager → 回复结果"的单段处理，与现有 `create` 同构。

### 3.3 参数校验

- `create`/`delete`：参数需 2 段且首段为子命令名，否则回复对应用法提示。
- `rename`：参数需 3 段（子命令、旧合集、新名称），否则回复用法提示。
- 新名称走 `validate_collection_name` 校验（与 `create` 相同规则：去首尾空白、禁内部空白/路径字符/`./..`/保留名"全局""全部合集"）。

### 3.4 插件层错误映射

| 异常 | 回复 |
|---|---|
| `CollectionNotFoundError` | "未找到表情包合集：{目标}\n发送 /switch 查看可用合集" |
| `CollectionNotEmptyError` | "合集不为空，请先 /move 或 /del 清空后再删除" |
| `CollectionRenameTargetExistsError` | "合集名称已存在：{name}（{id}）" |
| `CollectionPathConflictError` | create：`无法创建合集：同名路径不是可用目录"`；rename：`无法重命名：目标名称对应路径不是可用目录"` |
| `InvalidCollectionNameError` | 复用现有 `_INVALID_NAME` 文案 |
| `CollectionDeleteError` | "合集删除失败，请检查日志后重试" |
| `CollectionCreateError` | 仅 rename 复用（补偿失败）：`合集重命名失败，请检查日志后重试"`；create 沿用现有"合集创建失败，请检查日志后重试" |
| `RefreshInProgressError` | "索引正在刷新，请稍后再试" |
| `IndexAddCancelledError` | "服务正在关闭，请稍后再试" |
| `RuntimeError`（IndexManager 未就绪） | "服务未就绪，请稍后再试" |
| 其他未预期异常 | "合集删除/重命名失败，请检查日志后重试" + `logger.exception` |

## 4. MetadataStore 层改动

### 4.1 新增 `rename_collection`

```python
def rename_collection(self, collection_id: int, new_name: str) -> MemeCollection:
    ...
```

在 `self._lock` 内、一个 sqlite 事务里完成：

1. 校验 `collection_id > 0` 且合集存在于 `_id_to_collections`；否则抛 `ValueError`（调用方映射为 `CollectionNotFoundError`）。
2. `new_name` 非空且不等于原名的兜底校验；重复名称由 DB UNIQUE 约束兜底，抛 `sqlite3.IntegrityError`（调用方映射为 `CollectionRenameTargetExistsError`）。
3. `UPDATE meme_collection SET name = ? WHERE id = ?`。
4. **批量更新该合集所有 `meme.image_path` 首段**：每条 `image_path` 形如 `<旧名>/...`，改为 `<新名>/...`。逐条 `UPDATE meme SET image_path = ? WHERE id = ?`（条目数通常很少，无需批量优化）。`image_path` 全库 UNIQUE，重命名后只改首段，仍唯一，不会冲突。
5. `conn.commit()`；失败 `conn.rollback()` 并抛出，缓存不动。
6. 更新缓存：
   - `_id_to_collections[cid] = MemeCollection(cid, new_name)`；
   - `_collection_name_to_id` 删旧名 key、加新名；
   - `_entries` 与 `_entries_by_collection` 中该合集每条 frozen `MemeEntry` 重建为新实例（`image_path` 变了，frozen 不可原地改）。

返回新的 `MemeCollection(cid, new_name)`。

### 4.2 复用 `delete_collection_and_reset_scopes`

已有实现：空合集校验 + `BEGIN IMMEDIATE` 事务内回退所有引用该编号的 `chat_collection_scope` 到 0 + 删 `meme_collection` 行 + 缓存同步。**文件系统 rmdir 不在 store 层做**——保持"store 层只写 SQL/缓存，不碰文件系统"的现有边界。

### 4.3 不新增 store 层目录操作方法

空目录 `rmdir` 放在协调器层，store 层不碰文件系统。

## 5. 写编排层改动

### 5.1 `WriteOp` 枚举新增

```python
DELETE_COLLECTION = auto()
RENAME_COLLECTION = auto()
```

### 5.2 `_WriteRequest` 新增字段

```python
collection_id: int = 0          # DELETE_COLLECTION 用：目标合集编号
new_collection_name: str = ""   # RENAME_COLLECTION 用：已校验的新名称
```

`collection_name` 字段已存在（CREATE_COLLECTION 用），delete/rename 各取所需，互不冲突。

### 5.3 `_write_dispatch` 字典新增

```python
WriteOp.DELETE_COLLECTION: self._execute_delete_collection_guarded,
WriteOp.RENAME_COLLECTION: self._execute_rename_collection_guarded,
```

两个执行器均走 guarded 包装（与 `CREATE_COLLECTION` 一致：取得写锁后重检 `req.future.done()`，已取消则返回 None，外层不 `set_result`）。

### 5.4 `_execute_delete_collection`（写锁内，先文件后 DB）

顺序为**先 rmdir、后删 DB**，使 rmdir 失败时 DB 未动、无需回滚：

1. `req.future.done()` 检查 → 已取消返回 None。
2. `metadata_store.get_collection(req.collection_id)` → 不存在抛 `CollectionNotFoundError`。
3. **空合集校验**：`metadata_store.collection_entry_count(req.collection_id) > 0` → 抛 `CollectionNotEmptyError`。
4. **目录身份校验 + rmdir**：
   - `target = memes_dir / collection.name`；
   - 复用 `_get_collection_directory_identity(target)` 确认是普通目录（非符号链接、非文件），否则抛 `CollectionPathConflictError`；
   - `os.scandir` 复核目录为空（写锁内再验一次，防 DB 空判断与 rmdir 之间的并发写入）；
   - `target.rmdir()`；失败抛 `CollectionDeleteError`，**DB 未动**。
5. **再删 DB**：`metadata_store.delete_collection_and_reset_scopes(req.collection_id)`（写锁内再次确认空，事务内回退 ChatScope + 删行）。
   - 若这步失败（极少，sqlite 异常）：目录已删、DB 还在 → 补偿 `target.mkdir()` 恢复空目录，记 `logger.critical`，抛 `CollectionDeleteError`。
6. 返回 `DeleteCollectionResult(collection=collection, reset_scope_count=reset_count)`。

### 5.5 `_execute_rename_collection`（写锁内，先 SQLite 后文件）

1. `req.future.done()` 检查。
2. `metadata_store.get_collection(req.collection_id)` → 不存在抛 `CollectionNotFoundError`。
3. 重名检查：`metadata_store.get_collection_by_name(req.new_collection_name)` → 已存在抛 `CollectionRenameTargetExistsError`。
4. 目录身份与冲突校验：
   - `old_dir = memes_dir / old_name`，读 `old_identity = _get_collection_directory_identity(old_dir)`；`old_dir` 不存在、是文件/符号链接时该函数抛 `CollectionPathConflictError`（目录消失但 DB 记录仍在的极端场景，下次 `/refresh` 阶段2 才会清理，rename 拒绝并提示）；
   - `new_dir = memes_dir / new_name`，若 `new_dir` 已存在抛 `CollectionPathConflictError`。
5. `metadata_store.rename_collection(req.collection_id, req.new_collection_name)`（单事务改 name + 批量 image_path + 缓存）。
6. **文件系统重命名目录**：`old_dir.rename(new_dir)`（`memes/` 内部同盘，非跨 bind mount，`Path.rename` 安全）。
   - 若 rename 失败：SQLite 已改、目录未动 → 补偿：调 `metadata_store.rename_collection(req.collection_id, old_name)` 回滚 SQLite 与缓存；补偿失败记 `logger.critical`，抛 `CollectionCreateError`（复用，见 §5.7），回复"合集重命名失败，请检查日志后重试"。
7. 返回 `RenameCollectionResult(collection=MemeCollection(cid, new_name), old_name=old_name, new_name=new_name, entry_count=<受影响条目数>)`。

### 5.6 门面 `IndexManager` 新增两个薄入口

`delete_collection(raw: str) -> DeleteCollectionResult` 与 `rename_collection(raw: str, new_name: str) -> RenameCollectionResult`，结构与 `create_collection` 同构：

1. `if self._shutting_down: raise IndexAddCancelledError`；
2. `if self._refresh_active: raise RefreshInProgressError`；
3. 解析旧合集：`collection = await asyncio.to_thread(self._collection_manager.resolve_collection, raw)`（编号优先、名称兜底，已存在；不存在抛 `CollectionNotFoundError`）；
4. rename 的新名称：`validate_collection_name(new_name)`（非法抛 `InvalidCollectionNameError`）；
5. `self._ensure_write_worker()`；
6. 构造 `_WriteRequest`（delete 用 `collection_id`；rename 用 `collection_id` + `new_collection_name`）；
7. `return await self._coordinator.submit(req)`。

### 5.7 异常类型归属

- `CollectionNotFoundError`：已在 `collection_manager.py`，插件层与门面共用。
- `CollectionNotEmptyError`、`CollectionDeleteError`、`CollectionRenameTargetExistsError`：新增于 `index_types.py`，`RuntimeError` 子类。
- rename 的补偿失败复用现有 `CollectionCreateError`（语义同为"合集操作失败且可能需人工检查"），避免新增语义重叠的异常类；插件层对 rename 的 `CollectionCreateError` 回复"合集重命名失败，请检查日志后重试"。

## 6. 异常与结果类型定义

新增于 `index_types.py`：

```python
class CollectionNotEmptyError(RuntimeError):
    """待删除合集仍含表情包。"""

class CollectionDeleteError(RuntimeError):
    """删除合集失败且可能需要人工检查文件系统。"""

class CollectionRenameTargetExistsError(RuntimeError):
    """重命名目标名称已被其他合集使用。"""

    def __init__(self, collection: MemeCollection) -> None:
        self.collection = collection
        super().__init__(f"合集名称已存在: {collection.id}:{collection.name}")

@dataclass(frozen=True, slots=True)
class DeleteCollectionResult:
    """删除合集后的持久状态快照。

    Attributes:
        collection: 已删除的合集快照（用于回复编号与名称）。
        reset_scope_count: 回退到全部合集的 ChatScope 行数。
    """
    collection: MemeCollection
    reset_scope_count: int

@dataclass(frozen=True, slots=True)
class RenameCollectionResult:
    """重命名合集后的持久状态快照。

    Attributes:
        collection: 重命名后的合集快照（新名、同 id）。
        old_name: 重命名前的名称。
        new_name: 重命名后的名称。
        entry_count: 受 image_path 首段变更影响的条目数。
    """
    collection: MemeCollection
    old_name: str
    new_name: str
    entry_count: int
```

`bot/index_manager/__init__.py` 的 `__all__` 与 re-export 补齐：`CollectionNotEmptyError`、`CollectionDeleteError`、`CollectionRenameTargetExistsError`、`DeleteCollectionResult`、`RenameCollectionResult`。

## 9. 测试

### 9.1 MetadataStore 层

- `rename_collection` 改名成功：`meme_collection.name` 更新、该合集所有 `meme.image_path` 首段替换、缓存 `_id_to_collections`/`_collection_name_to_id`/`_entries`/`_entries_by_collection` 同步、其他合集条目不受影响。
- `rename_collection` 重名 → `IntegrityError`，缓存与 DB 不变。
- `rename_collection` 不存在 id → 抛 `ValueError`。
- `delete_collection_and_reset_scopes` 已有测试，确认非空仍拒绝。

### 9.2 协调器/门面层

- `delete_collection`：空合集 → DB 删 + 目录 rmdir + ChatScope 回退；非空 → `CollectionNotEmptyError` 且 DB/目录不动；不存在 → `CollectionNotFoundError`；rmdir 失败（mock 目录非空/身份变化）→ DB 未动、抛 `CollectionDeleteError`；删 DB 失败（mock）→ 补偿 mkdir、抛 `CollectionDeleteError`。
- `rename_collection`：成功 → DB name + image_path 首段 + 目录重命名；目标名已存在 → `CollectionRenameTargetExistsError` 且不动；目录 rename 失败（mock）→ SQLite 回滚到旧名、抛 `CollectionCreateError`；refresh 进行中 → `RefreshInProgressError`；shutting_down → `IndexAddCancelledError`。
- 写锁互斥：delete/rename 与 refresh/add 经 Write Worker 串行（沿用现有并发测试模式）。

### 9.3 插件层

- `/collection delete <编号>`、`/collection rename <旧> <新>` 成功回复文案（含 reset_scope_count 追加行、entry_count 行）。
- 群聊 → "此命令仅限私聊使用"；非授权 → 静默；参数缺失 → 用法提示；会话互斥 → "已有命令在处理中，请先 /cancel"。
- 各错误分支回复文案（`CollectionNotFoundError`、`CollectionNotEmptyError`、`CollectionRenameTargetExistsError`、`CollectionPathConflictError`、`InvalidCollectionNameError`、`RefreshInProgressError`、`IndexAddCancelledError`、未就绪）。

### 9.4 一致性核对

- `WriteOp` 枚举、`_write_dispatch` 字典、`_WriteRequest` 字段、`__init__.py` re-export 四处同步。
- rename 不动 chroma（`collection_id` 不变）——测试断言 `vector_store` 无 `update_collection_id` 调用。
- rename 不影响 ChatScope（`collection_id` 不变）——测试断言 `chat_collection_scope` 行不变。
- delete 回退 ChatScope 复用现有 `delete_collection_and_reset_scopes`，行为已验证。
- `validate_collection_name` 对 rename 新名称复用，保留名、内部空白、路径字符、`.` 开头等同样拒绝。
