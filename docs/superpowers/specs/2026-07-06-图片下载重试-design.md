# 图片下载重试机制设计

> 日期：2026-07-06  
> 状态：已批准，待实现

---

## 1. 背景

`/add` 命令在私聊中添加表情包时，需要通过 QQ 图片 URL 下载用户发送的图片。当前实现（`bot/plugins/meme_add.py` 的 `_download_image`）使用 `httpx.AsyncClient.get` 单次请求，30 秒超时，没有任何重试机制。网络抖动、QQ 图片服务器临时 5xx 等问题会导致添加失败，影响用户体验。

## 2. 目标

为 `/add` 图片下载增加重试机制，提高添加成功率，同时保持失败行为清晰、不引入过度重试。

## 3. 约束

- 复用项目现有 `bot/engine/retry_config.py` 的 `api_retry` 默认参数（最多 3 次尝试、指数退避 1–10 秒）。
- 仅对网络/超时异常和 HTTP 5xx 服务器错误重试。
- HTTP 4xx 客户端错误直接失败，不重试。
- 保持现有 30 秒下载超时不变。
- 复用 `api_retry` 的 `before_sleep_log` 日志输出。

## 4. 设计方案

采用方案 A：直接装饰 `_download_image`。

### 4.1 修改文件

`bot/plugins/meme_add.py`

### 4.2 具体改动

1. 从 `bot.engine.retry_config` 导入 `api_retry`。
2. 在模块内新增自定义异常：

   ```python
   class DownloadServerError(RuntimeError):
       """QQ 图片服务器返回 5xx，允许重试。"""
   ```

3. 为 `_download_image` 添加装饰器：

   ```python
   @api_retry(extra_exceptions=(DownloadServerError,))
   async def _download_image(url: str) -> tuple[bytes, httpx.Response]:
       ...
   ```

4. 函数内部错误分类逻辑：

   ```python
   async with httpx.AsyncClient() as client:
       response = await client.get(url, timeout=DOWNLOAD_TIMEOUT, follow_redirects=True)
       if response.status_code >= 500:
           raise DownloadServerError(
               f"图片服务器错误 {response.status_code}: {url}"
           )
       response.raise_for_status()
       return response.content, response
   ```

### 4.3 重试行为

| 异常/状态 | 是否重试 | 说明 |
|-----------|----------|------|
| `httpx.NetworkError` / `ConnectError` / `TimeoutException` | 是 | `api_retry` 默认捕获 |
| `ConnectionError` / `TimeoutError` | 是 | `api_retry` 默认捕获 |
| HTTP 5xx | 是 | 手动抛 `DownloadServerError`，通过 `extra_exceptions` 触发重试 |
| HTTP 4xx | 否 | `response.raise_for_status()` 直接抛 `httpx.HTTPStatusError`，不在重试范围内 |

### 4.4 日志

`api_retry` 内置 `before_sleep_log(logger, logging.WARNING)`，每次重试前会自动输出 WARNING 日志，包含异常信息和重试间隔，无需额外实现。

## 5. 影响范围

- 仅影响 `bot/plugins/meme_add.py` 中的 `_download_image`。
- `/add` 命令的图片下载失败率预计降低。
- 不改变 `/add` 其他流程（压缩、OCR、embedding、写入索引）。
- 不引入新的环境变量或配置项。

## 6. 测试考虑

- 单元测试：使用 `respx` 或 `httpx.MockTransport` 模拟 502/503/504，验证会重试；模拟 404/403，验证不重试。
- 单元测试：模拟 `httpx.ConnectError`，验证会重试。
- 可测试性：`_download_image` 本身无状态，仅依赖 URL 和 `DOWNLOAD_TIMEOUT`，容易 mock。

## 7. 风险

- 极端情况下，QQ 图片服务器持续 5xx 会导致最多 3 次重试、总等待时间最长约 30 秒（单次超时）+ 20 秒（退避）≈ 50 秒，仍在合理范围。
- 该风险已通过限制重试次数和退避上限得到控制。
